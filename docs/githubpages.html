🚀 GitHub Pages Deployment Setup

Repository: https://github.com/tgkprog/trafficInRain

Expected URL: https://tgkprog.github.io/trafficInRain/

ℹ️ What This Does:

This setup will automatically build and deploy your Traffic in Rain simulation to GitHub Pages every time you push code to the main branch.

📋 Prerequisites

  • GitHub repository: tgkprog/trafficInRain (already exists)
  • Code pushed to main branch
  • GitHub account with access to repository settings

🔧 One-Time GitHub Settings Configuration

Step 1:

Enable GitHub Pages

  1. Go to your repository: https://github.com/tgkprog/trafficInRain
  2. Click on Settings tab (⚙️ gear icon at the top)
  3. In the left sidebar, scroll down and click on Pages
  4. Under "Build and deployment" section:
    • Source: Select GitHub Actions from the dropdown
📸 Settings → Pages → Source: "GitHub Actions"
⚠️ Important: Make sure you select "GitHub Actions" as the source, NOT "Deploy from a branch". This allows the workflow to deploy automatically.
Step 2:

Verify Workflow Permissions

  1. Still in repository Settings
  2. In the left sidebar, click on Actions → General
  3. Scroll down to "Workflow permissions"
  4. Ensure "Read and write permissions" is selected
  5. Ensure "Allow GitHub Actions to create and approve pull requests" is checked
  6. Click Save if you made changes
📸 Settings → Actions → General → Workflow permissions: "Read and write permissions"

📦 Files Already Added to Your Repository

✅ The following files have been automatically created:

1. GitHub Actions Workflow

File: .github/workflows/deploy.yml

This workflow will:

  • Trigger on every push to main branch
  • Install Node.js 20
  • Install dependencies with npm ci
  • Build the project with npm run build
  • Deploy the dist/ folder to GitHub Pages

2. Vite Configuration Update

File: vite.config.js

Added base: '/trafficInRain/' to ensure correct asset paths on GitHub Pages.

🚀 Deployment Process

Step 3:

Push Your Code

Once you've completed Steps 1 & 2 on GitHub, push your code:

git add .
git commit -m "Add GitHub Pages deployment workflow"
git push origin main
Step 4:

Monitor the Deployment

  1. Go to your repository on GitHub
  2. Click on the Actions tab
  3. You should see a workflow run called "Deploy to GitHub Pages"
  4. Click on it to see the build and deployment progress
  5. Wait for both "build" and "deploy" jobs to complete (usually 1-2 minutes)
📸 Actions tab → "Deploy to GitHub Pages" workflow running
🎉 Success! Once the workflow completes, your site will be live at:

https://tgkprog.github.io/trafficInRain/

🐛 Troubleshooting

Problem: Workflow fails with "permission denied"

Solution: Check Step 2 above - ensure workflow permissions are set to "Read and write permissions"

Problem: Site shows 404 or blank page

Possible causes:

  • Wait a few minutes after first deployment
  • Clear browser cache (Ctrl+Shift+R or Cmd+Shift+R)
  • Check that base: '/trafficInRain/' is in vite.config.js
  • Verify the workflow completed successfully in the Actions tab

Problem: Assets not loading (CSS, JS missing)

Solution: Verify vite.config.js has base: '/trafficInRain/' configured

Problem: Workflow doesn't trigger

Checklist:

  • Ensure you pushed to the main branch (not master or another branch)
  • Check that .github/workflows/deploy.yml exists in your repository
  • Go to Settings → Actions → General and ensure Actions are enabled
