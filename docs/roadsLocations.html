Data Extraction Guide

Roads & Locations

Step-by-step instructions for extracting realistic map data from Google Maps to build the simulation's road network and location coordinates.

⓪ Overview

What We Need to Extract

To render a realistic simulation, we need real-world GPS coordinates, road paths, road widths, and routing data from the Vishrantwadi & Viman Nagar area.

📍 Location Data

  • GPS coordinates (lat/lng) for all 9 places
  • Place type (residential, commercial, school)
  • Entry/exit point on nearest road

🛤️ Road Data

  • Waypoints along each road path
  • Road widths (narrow / medium / wide)
  • Number of lanes & direction

🗺️ Network Data

  • Intersections / junction nodes
  • Connectivity graph between locations
  • One-way restrictions (if any)
🔧 Tools Needed

What You'll Use

All free tools — no paid APIs required for initial data extraction.

🌍 Google Maps (Primary)

  • Google Maps web — search places, right-click for coordinates
  • Google My Maps — create custom map with pins & paths
  • Measure distance tool — right-click → "Measure distance"
  • Street View — verify road widths visually
↗ Google Maps ↗ Google My Maps

🗺️ OpenStreetMap (Alternative)

  • Overpass Turbo — query OSM data for roads & buildings
  • JOSM editor — inspect road widths, lanes, surface
  • Geojson.io — draw shapes, export as GeoJSON
  • OSM often has lane counts & road widths tagged
↗ Overpass Turbo ↗ geojson.io
💡

Recommended approach: Use Google My Maps to pin all 9 locations and draw roads between them, then export as KML. Supplement with Overpass Turbo for lane/width metadata.

2 Step 2

Trace Road Paths Using Google My Maps

Create a custom map with all road segments drawn as polylines to capture realistic road geometry.

2a

Create a New Google My Maps Project

This will be your master reference map
  • Go to Google My Maps and click "+ Create a new map"
  • Name it "Traffic in Rain — Vishrantwadi & Viman Nagar"
  • Create a layer called "Locations"
  • Create another layer called "Roads"
2b

Pin All 9 Locations

In the "Locations" layer
  • Select the "Locations" layer
  • Click the pin/marker icon in the toolbar (🔍 or marker)
  • Search for each place and drop a pin at its entrance
  • Name each marker (e.g., "Aero Mall", "Planet Medical")
  • Color-code them by type: 🟢 Residential, 🔵 Commercial, 🟡 School
2c

Draw Road Polylines

In the "Roads" layer — this captures road geometry as waypoints

For each road segment connecting two locations:

  • Select the "Roads" layer
  • Click the "Draw a line" tool → choose "Add line or shape"
  • Click along the centre of the road, following its curves and bends
  • Place a point at every bend or curve — these become your waypoints
  • Place points every ~20–30 metres on straight sections
  • Double-click to finish the line
  • Name the line (e.g., "Airport Rd: Aero Mall → Vibgyor")
  • Color wide roads green, medium blue, narrow yellow
💡

Switch to Satellite view while drawing roads — it's much easier to see the actual road edges and curves compared to the default map view.

2d

Export as KML

This gives you all coordinates in a structured file
  • Click the ⋮ (three dots) menu next to the map title
  • Select "Export to KML/KMZ"
  • Check "Export as KML instead of KMZ"
  • Download and save as map-data.kml in your project's docs/ folder
ℹ️

KML is XML with <coordinates> tags containing longitude,latitude,altitude triplets. We'll parse these into JSON in Step 4.

3 Step 3

Measure Road Widths & Lane Counts

Use Google Maps' measure tool and Street View to determine how wide each road segment is.

3a

Measure with "Measure Distance" Tool

Get approximate road widths in metres
  • On Google Maps (satellite view), right-click on one edge of the road
  • Select "Measure distance"
  • Click on the opposite edge of the road
  • Read the distance — this is the road width
  • Record it for each road segment
⚠️

Zoom in to maximum level in satellite view for accurate edge-to-edge measurement. At low zoom, you could be off by several metres.

3b

Verify with Street View

Visual confirmation of lane counts
  • Drag the yellow pegman onto each road segment
  • Count the number of lanes visible
  • Note if there are medians, dividers, or parking on the side
  • Check if roads narrow at certain points (pinch points cause congestion!)
  • Note any one-way restrictions visible on signs
3c

4 Step 4

Convert Extracted Data to Project JSON

Transform your Google Maps data into the config JSON files the simulation engine expects.

4a

Parse KML to Extract Coordinates

Use this Node.js helper script

Save this script as scripts/parse-kml.js in your project and run it on the exported KML file.

// scripts/parse-kml.js // Usage: node scripts/parse-kml.js docs/map-data.kml const fs = require('fs'); const kmlFile = process.argv[2] || 'docs/map-data.kml'; const kml = fs.readFileSync(kmlFile, 'utf-8'); // Extract Placemarks (locations & roads) const placemarks = kml.match(/<Placemark>[\s\S]*?<\/Placemark>/g) || []; const locations = []; const roads = []; placemarks.forEach(pm => { const name = pm.match(/<name>(.*?)<\/name>/)?.[1] || 'Unnamed'; const coordsRaw = pm.match( /<coordinates>([\s\S]*?)<\/coordinates>/ )?.[1]?.trim(); if (!coordsRaw) return; const points = coordsRaw.split(/\s+/).map(c => { const [lng, lat] = c.split(',').map(Number); return { lat, lng }; }); if (points.length === 1) { // Single point = location marker locations.push({ name, ...points[0] }); } else { // Multiple points = road polyline roads.push({ name, waypoints: points }); } }); console.log('=== LOCATIONS ==='); console.log(JSON.stringify(locations, null, 2)); console.log('\n=== ROADS ==='); console.log(JSON.stringify(roads, null, 2));
4b

Save to src/config/locations.json

5 Step 5

Normalize GPS to Canvas Coordinates

Convert real-world lat/lng into pixel positions on the simulation canvas.

5a

The Projection Formula

Linear mapping from GPS bounding box to canvas pixels

Since the area is small (~1.5 km²), a simple linear projection works. No need for Mercator.

// src/utils/projection.js // Bounding box of all locations (find min/max from your data) const BOUNDS = { minLat: 18.5580, // southernmost point maxLat: 18.5710, // northernmost point minLng: 73.9040, // westernmost point maxLng: 73.9200, // easternmost point }; const CANVAS_WIDTH = 1400; // pixels const CANVAS_HEIGHT = 900; // pixels const PADDING = 60; // edge padding in pixels export function gpsToCanvas(lat, lng) { // Normalize to 0–1 range const nx = (lng - BOUNDS.minLng) / (BOUNDS.maxLng - BOUNDS.minLng); const ny = (lat - BOUNDS.minLat) / (BOUNDS.maxLat - BOUNDS.minLat); // Map to canvas pixels (flip Y: lat increases north, canvas Y increases down) const x = PADDING + nx * (CANVAS_WIDTH - 2 * PADDING); const y = PADDING + (1 - ny) * (CANVAS_HEIGHT - 2 * PADDING); return { x: Math.round(x), y: Math.round(y) }; } // Convert real-world meters to canvas pixels export function metersToPixels(meters) { // ~111,320 m per degree latitude at this latitude const degreesPerMeter = 1 / 111320; const latRange = BOUNDS.maxLat - BOUNDS.minLat; const pixelsPerDegree = (CANVAS_HEIGHT - 2 * PADDING) / latRange; return Math.round(meters * degreesPerMeter * pixelsPerDegree); }
5b

Batch Convert All Coordinates

Script to generate canvas-ready config

Run this after you have final locations.json and roads.json with GPS coordinates:

// scripts/convert-to-canvas.js // Usage: node scripts/convert-to-canvas.js const { gpsToCanvas, metersToPixels } = require('../src/utils/projection'); const locations = require('../src/config/locations.json'); const roads = require('../src/config/roads.json'); // Convert locations const canvasLocations = locations.map(loc => ({ ...loc, ...gpsToCanvas(loc.lat, loc.lng), // adds x, y })); // Convert road waypoints + widths const canvasRoads = roads.edges.map(road => ({ ...road, widthPixels: metersToPixels(road.widthMeters), waypoints: road.waypoints.map( wp => gpsToCanvas(wp.lat, wp.lng) ), })); console.log(JSON.stringify({ canvasLocations, canvasRoads }, null, 2));
💡

Result: Your locations get x, y pixel positions. Your road waypoints become pixel coordinates. Road widths become pixel widths. The simulation renderer uses these directly.

⊕ Alternative Method

Extract Roads from OpenStreetMap

OSM has detailed road data including lane counts and surface types. Use Overpass Turbo to query it programmatically.

A1

Overpass Turbo Query

Get all roads in the Vishrantwadi–Viman Nagar bounding box

Go to Overpass Turbo and paste this query:

// Overpass QL — roads in Vishrantwadi–Viman Nagar area [out:json][timeout:25]; ( // Get all highways (roads) in bounding box way["highway"] (18.557,73.903,18.572,73.921); ); out body; >; out skel qt;
  • Click "Run" to execute the query
  • Roads will be highlighted on the map
  • Click "Export" → "GeoJSON" to download
  • Save as docs/osm-roads.geojson
A2

What OSM Road Data Includes

Much richer metadata than Google Maps
OSM Tag What It Tells You Useful For

lanesNumber of lanesDirect input to road config
widthRoad width in metresDirect input — no measuring needed!

onewayOne-way restrictionRouting constraints
nameRoad nameDisplay labels
ℹ️

Note: Not all OSM roads have lanes and width tagged. For those that don't, fall back to your Google Maps measurements from Step 3.

A3

Parse GeoJSON to Project Config

Convert OSM export to roads.json format
// scripts/parse-osm-geojson.js // Usage: node scripts/parse-osm-geojson.js docs/osm-roads.geojson const fs = require('fs'); const geo = JSON.parse( fs.readFileSync(process.argv[2], 'utf-8') ); const roads = geo.features .filter(f => f.geometry.type === 'LineString') .map(f => ({ name: f.properties.name || 'Unnamed Road', type: f.properties.highway, lanes: parseInt(f.properties.lanes) || 2, width: parseFloat(f.properties.width) || null, oneway: f.properties.oneway === 'yes', waypoints: f.geometry.coordinates.map( ([lng, lat]) => ({ lat, lng }) ), })); console.log(JSON.stringify(roads, null, 2));
