1 Technology Stack

Node.js Frontend with a 2D Physics Engine

A Vite-powered project using an HTML5 Canvas renderer and Matter.js for realistic vehicle physics and collision handling.

⚡ Build Tool

  • Vite — fast HMR dev server, production bundling
  • Vanilla JS (or TypeScript optional)
  • No heavy framework; keep it lightweight

🔩 Physics Engine

  • Matter.js — 2D rigid body physics
  • Handles collision, friction, velocity
  • Rain modifies friction coefficients & max speeds

🎨 Rendering

  • HTML5 Canvas (2D context)
  • Custom renderer for roads, vehicles, rain
  • Layered drawing: map → vehicles → weather overlay

📦 Key npm Packages

  • matter-js — physics simulation
  • pathfinding — A* route computation
  • dat.gui or lil-gui — real-time controls
  • stats.js — FPS monitoring

🗂️ Dev Dependencies

  • vite — bundler & dev server
  • eslint — code linting
  • prettier — formatting
  • Optional: vitest — unit tests

High-Level System Flow

A modular design separating configuration, simulation engine, rendering, and UI layers.

🧩 Core Modules

  • SimulationManager — orchestrates tick loop, spawning, removal
  • VehicleFactory — creates entities from config
  • RoadNetwork — graph of nodes & edges with lane widths
  • PathPlanner — A* routing with traffic-aware weights
  • WeatherController — applies rain modifiers globally

🔄 Simulation Loop

  • 1. Spawn vehicles at origin locations (configurable rate)
  • 2. Each vehicle picks a destination & computes route
  • 3. Steering behaviour follows road waypoints
  • 4. Matter.js resolves physics each tick (60 Hz)
  • 5. Remove vehicles that reach destination
  • 6. Renderer draws updated state + rain overlay

6 Physics Engine Details

Matter.js Configuration

How the physics engine models vehicle movement, collisions, and rain effects on road surfaces.

⚙️ Engine Settings

  • Tick rate: 60 Hz (16.67 ms per step)
  • Gravity: disabled (top-down 2D view)
  • Position iterations: 6 (collision accuracy)
  • Velocity iterations: 4
  • Constraint iterations: 2
  • Custom beforeUpdate event for steering forces

🚦 Vehicle AI (Steering)

  • Seek: steer toward next waypoint on route
  • Arrive: decelerate approaching destination
  • Separation: avoid overlapping nearby vehicles
  • Lane keeping: tendency to stay in lane centre
  • Speed compliance: obey road speed limits
  • Queue: slow behind slower vehicle ahead

7 Weather Model

Three Conditions Explained

Each weather mode modifies physics, visuals, and traffic patterns to demonstrate why rain causes congestion.

☀️

No Rain

  • Full speed for all vehicles
  • Normal friction on roads
  • Maximum pedestrian & cycle traffic
  • Clear canvas — no overlay
  • Baseline for comparison
🌦️

Light Rain

  • 10–30% speed reduction across types
  • Reduced friction → longer braking
  • Fewer pedestrians & cyclists on roads
  • Animated rain drops (sparse, slow)
  • Mild waterlogging on narrow roads
🌧️

Heavy Rain

  • 20–60% speed reduction
  • Greatly reduced friction & visibility
  • Almost no pedestrians/cyclists
  • Dense rain animation + fog overlay
  • Severe waterlogging blocking narrow roads

💡 Why Traffic Slows in Rain — Factors Modelled

  • Reduced tyre grip — lower friction means drivers must go slower to maintain control
  • Poor visibility — vehicles must keep larger following distances
  • Waterlogged roads — narrow lanes become partially blocked, creating single-lane bottlenecks
  • Mode shift — fewer pedestrians & cyclists; more autos & cars → more vehicles occupy more road space
  • Slower acceleration — cautious starts at intersections propagate delays (shockwave)
  • Increased following distance — effective road capacity drops even when speeds are similar
8 UI & Controls

Interactive Dashboard

Real-time controls for weather, simulation speed, vehicle mix, and live traffic metrics.

🎛️ Control Panel (lil-gui)

  • Weather toggle: No Rain / Light / Heavy (radio buttons)
  • Simulation speed: 0.5× – 5× slider
  • Vehicle density: Low / Medium / High
  • Vehicle type toggles: enable/disable each of the 12 types
  • Show routes: overlay pathfinding debug lines
  • Pause / Reset

📊 Live Metrics Overlay

  • Active vehicles on the map (count)
  • Average speed across all vehicles
  • Average trip time (origin → destination)
  • Road congestion heat-map (colour-coded edges)
  • FPS counter (stats.js)
  • Throughput — vehicles reaching destination / min

🖼️ Canvas Layout

The canvas fills most of the viewport with a floating control panel on the right. Layers are drawn in order:

  1. Layer 0 — Background: Solid dark ground with road grid texture
  2. Layer 1 — Roads: Road surfaces with lane markings drawn from graph data
  3. Layer 2 — Locations: Named markers for the 9 places
  4. Layer 3 — Vehicles: Coloured rectangles sized to scale, rotated to heading
  5. Layer 4 — Weather: Rain particles + fog overlay (transparent fill)
  6. Layer 5 — HUD: Stats, metrics, labels
